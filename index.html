Indicaciones para las Aplicaciones

En todas las aplicaciones se deben incluir las referencias a las funcionalidades básicas (Javascript) de comportamiento de cada uno de los componentes y las hojas de estilos que están actualmente disponibles en servidores ACHS. Para ello copie las siguientes líneas en su aplicación

Consejos para garantizar el buen comportamiento de los Componentes

  • Los ID deben ser únicos. NO DEBEN REPETIRSE nunca
  • La estructura del HTML base de cada uno de los componentes debe mantenerse. Es decir, debe respetarse, herencia de elementos en el DOM, ids, y clases.
  • La apariencia general de las ventanas debe mantenerse, es decir, no deben hacerse cambios de estilo, tamaño de fuentes, colores etc. La idea es mantener la consistencia gráfica entre todas las aplicaciones ACHS.
  • Las funciones javascript que intervienen en los componentes (librería controles-achs.js), no debe ser modificada. Ya que al cambiar el comportamiento de los controles se pierde la línea del funcionamiento general de los componentes ACHS.

	$('.boxCalendar, .popCalendar').click(
		function(event){

    	// Para definir un rango de fecha
      	calendarioAbierto.datepicker( "option", "minDate",' -3d' ); // Si desea definir un rango de fecha mínima utilice esta sentencia
      	calendarioAbierto.datepicker( "option", "maxDate", '+2d' ); // Si desea definir un rango de fecha máxima utilice esta sentencia

      // Para definir un rango de años
      	calendarioAbierto.datepicker( "option", "yearRange", '2000:2010' ); // Si desea definir un rango de años utilice esta sentencia

    	}
	);
 });

Las propiedades de configuración que se pueden agregar al calendario son:

minDate: establece el límite anterior a la fecha actual
maxDate: establece el límite superior a la fecha actual
yearRange: establece el límite de años que desee mostrar en el calendario

Para calcular el rango de fecha se utilizan los operadores matematicos + y - para sumar o restar días, meses, años y semanas, la forma de utilizarlo es la siguiente:

+-< NUMERO > d (sumar o restar días)
+-< NUMERO > m (sumar o restar meses)
+-< NUMERO > y (sumar o restar años)
+-< NUMERO > w (sumar o restar semanas)

Ejemplos:

minDate: '-3d', maxDate: '0' (Permite seleccionar una fecha: desde 3 días antes a la fecha actual, hasta la fecha actual)

minDate: '0', maxDate: '+12d' (Permite seleccionar una fecha: desde la fecha actual, hasta la fecha actual más 12 días)

minDate: '-1d', maxDate: '+1m +4d' (Permite seleccionar una fecha: desde un día menos a la fecha actual, hasta la fecha actual más 1 mes y 4 días)


Para poner el rango de años basta con definir un año de inicio y otro de fin

Ejemplo:
yearRange", '2000:2010' (Define que sólo se mostrará en el calendario los años entre el 2000 y 2010)

Sin embargo el calendario por defecto viene con un rango relativo de 65 años hacia atrás y 65 años hacia adelante, si desea cambiar este rango utilice en vez de años números enteros representando valores relativos

Ejemplo:
yearRange", '-50:+50' (Define que siempre se mostrarán 50 años antes y 50 años después del año actual)
